Project Structure

Here is a quick overview of how a vader project is structured.

Directories

Vaderjs manages asset linking for you relative to your sites url, this makes it easier for devs to link assets without having to make sure they escaped folders properly, or having to use direct absolute paths. Below is a quick overview of the directories vader looks for at the root of your project.

- src/* - Your project source code (components, views, styles, etc.) - public/* - Assets that you want to be publicly accessible (images, fonts, etc.)

src/
The src/ folder is where most of your project source code lives. This includes: - components/* - Reusable components - views/* - Page views seen by the user - pages/* - Logic for handling page views
public/
The public/ folder is where most of your project source code lives. This includes: - images/* - Images for your site - fonts/* - Fonts for your site - js/* - External js libraries you want to import
**Tip:**

You can turn off relative file paths using the
vader-disable_relative-paths \n comment in your views and main index file
